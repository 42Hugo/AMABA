GUI documentation

The Graphical user interface is made with the library customtkinter (see documentation here: https://customtkinter.tomschimansky.com/) The GUI will interact with both the printer object and the pneumatic object.

Initialisation

amabaGUI.__init__(pneumatic, printer)[source]

constructor method that also implements every GUI elements: buttons, frames, entry… the methods relative to the GUI are defined later in the code

Parameters:

  • pneumatic (object) – shared pneumatic object betwen the class GUI and printer to both update the pneumatic elements

  • printer (object) – object to interact with the printer variables and methods

  • raspberryPi (int) – varible to specify is currently running on the raspberry pi with the toutch screen, if 1 the mouse won’t be visible and full screen will be on

  • filePath (str) – path to the g-code file to print

  • locked – state of the GUI, locked is equal to 1 when a print is ongoing to prevent specific buttons to be activated, defaults to 0

GUI specific Methods

amabaGUI.lock_gui()[source]

allows to lock or unlock the GUI to prevent certain actions by the user during a print

amabaGUI.send_loop()[source]

Loop occuring every 500ms to update the GUI

amabaGUI.gcodeF()[source]

opening the g-code frame with options to start g-code from a file. First connection with the printer is started and a homing is made, be careful homing can’t be cancelled

amabaGUI.test_depose()[source]

opening the test depose frame with options to draw lines without a file. First connection with the printer is started and a homing is made, be careful homing can’t be cancelled

amabaGUI.test_substrat()[source]

opening the g-test substrat frame with options to test a substrat with multiple line print without a file. First connection with the printer is started and a homing is made, be careful homing can’t be cancelled

amabaGUI.onF(case)[source]

function to control the 3 solenoid valves from the GUI with switch

Parameters:

case (str) – string to indicate which switch was activated, ‘p’ for pointeau (nozle), ‘c’ for cartouche (pressure on the fluid), ‘a’ for atomization (spaying pressure)

amabaGUI.sliderFc(value)[source]

method to update the label of the cartouche based on the slider

Parameters:

value (float) – value between 0 and 1.7 bar with 2 digits

amabaGUI.sliderFa(value)[source]

method to update the label of the atomization based on the slider

Parameters:

value (float) – value between 0 and 2 bar with 2 digits

amabaGUI.update()[source]

update every label and slider according to the current values on the pneumati elements

amabaGUI.choose_substrat(sub)[source]

called by the option menu of the substrat, allows to use predefned substrat characteristics new substrat can be added with a new elif with the name as well as adding the new name to the option “substrat_options””
