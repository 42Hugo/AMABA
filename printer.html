  • pneumatic – Shared pneumatic object between the class GUI and printer to both update the pneumatic elements

  • flag (int) – Flag to indicate “ok” received from the printer, printer sendsok when receiing any g-code line and when receiving M400 waits for the end of movement before sending ok, defaults to 0

  • depose (int) – printing state, on (1) or off(0), defaults to 0

  • p (object) – Placeholder for the printer object, defaults to None

  • current_response (str) – Stores the current response from the printer

  • gcode_lines (str) – Stores the G-code lines to be sent to the printer

  • homed (int) – Indicates if the printer has done homing since boot, defaults to 0

  • speed (int) – Printing speed, defaults to 1800

  • z (int) – Printing height not including printing bed, defaults to 1

  • ydist (int) – Start y position for substrate test, defaults to 15

  • line (int) – current position when Moving y position with button next line and prev position, defaults to ydist

  • sample_size_x (int) – Sample size in x-direction, defaults to 70

  • sample_size_y (int) – Sample size in y-direction, defaults to 120

  • line_space (int) – Space between lines drawen with test substrat button or next/prev position, defaults to 10

  • x_space_on_borders (int) – Space left without print when testing lines on each side, defaults to 5

  • sub (int) – thickness of the substrate, 0 if no substrate, defaults to 6 (PMMA)

  • min_z (float) – Minimum z height, defaults to 0.1

  • max_z (float) – Maximum z height, defaults to 20

  • min_sub (int) – Minimum substrate height, defaults to 0

  • max_sub (int) – Maximum substrate height, defaults to 50

  • min_speed (int) – Minimum printing speed, defaults to 500

  • max_speed (int) – Maximum printing speed, defaults to 12000

  • bed_max_y (int) – Maximum y-dimension of the print bed, defaults to 250

  • bed_max_x (int) – Maximum x-dimension of the print bed, defaults to 210

  • z_offset (float) – Offset in z-direction, taken as a negative value to verify and change if needed, defaults to 1.3

  • stop (int) – Flag to stop the printing process, defaults to 0

  • on_going_print (int) – Indicates if a print is currently ongoing, defaults to 0

  • multilayer_print (int) – Indicates if a multilayer print from g-code is ongoing, defaults to 0

  • go_next_layer (int) – Flag to proceed to the next layer, changed by a timer or user input, defaults to 0

  • mutex (threading.Lock) – Lock to manage concurrent access with reading and writting on piles

  • pile_gcode (list) – Queue of G-code instructions to be sent (stack by group with the same depose state, printing or movement)

  • pile_change_layer (list) – Queue of layer change, 1 if after this layer time is needed to dry, defaults to 0

  • pile_depose (list) – Queue of depos instructions, 0 mean movement without printing and 1 nozzle, fluid and atomization on, defaults to 0

  • thread_state (int) – State of the thread, defaults to 1 (running)

  • wait_minutes (int) – Wait time between layers in minutes, defaults to 60

Gcode handling

The library printrun is used to handle the comunication with the printer including the serial comunication. an object called p is created and is linked to the printer. calling ‘self.p.send_now(‘G1 X1 Y10’)’ is a function provided by the library to inject g-code to the printer and it is the method used to handle the g-code injection

Printer.get_line_and_modify()[source]

the gcode lines that have to be sent are first checked here and modified if necessary

Notes on G-code lines used:

  • M400: This command causes G-code processing to pause and wait in a loop until all moves in the planner are completed, sends okay when done. This command is used to synchronize the pneumatic with the printer.

  • M862:Perform an axis continuity test for position encoder modules

  • M104: Set Hotend Temperature

  • M109: Wait for Hotend Temperature

  • G28: auto home

  • G80: Cancel Current Motion Mode - in the start g-code it also adds the mesh be leveling

  • G1 Z0.2/0.3 F720 : these are the 2 standard oprtions for the intro line of the mk3, they can be deleted from the slicer directly (printer->g-code personalisé -> g-code start)

Most movments starts with G1 and E stands for extruder, when positive printing is ongoing, when negative the print will stop. when there is no E it is a simple movment

Parameters:

  • depose – 0 if not currently printing, 1 otherwise, defaults to 0

  • change_depose_state (int) – 0 if the depose state is same as the previosu g-code commande, 1 otherwise, defaults to 0

  • layer – list of g-code lines with the same depose state

  • change_layer (int) – 1 when the current group of command is the last before change of layer, defaults to 0

  • print_data (list) – list with 3 informations, a list of g-code commands all having the same depose state, the information if the change of layer is happening. print_data=[layer,depose, change_layer]
